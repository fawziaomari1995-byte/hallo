## 2. ملف قاعدة البيانات (db.js)

### الوظيفة
إعداد واتصال قاعدة بيانات SQLite المشتركة عبر التطبيق

### الكود:
```javascript
const sqlite3 = require('sqlite3').verbose();

const dbDir = path.join(__dirname, 'db');
if (!fs.existsSync(dbDir)) {
  fs.mkdirSync(dbDir, { recursive: true });  // إنشاء المجلد إذا لم يكن موجوداً
}

const dbFile = path.join(dbDir, 'events.db');
const db = new sqlite3.Database(dbFile, (err) => {
  if (err) {
    console.error('Unable to open database:', err.message);
    process.exit(1);  // إيقاف التطبيق عند فشل الاتصال
  }
});

module.exports = db;  // تصدير الاتصال للاستخدام في الملفات الأخرى
```

### النقاط المهمة:
- تشغيل Verbose لعرض رسائل الأخطاء التفصيلية
- الاتصال يبقى مفتوحاً طوال حياة التطبيق
- تُستخدم نفس الاتصالة في جميع العمليات

  (req, res, next) => {
    const authHeader = req.headers.authorization || '';
    const token = authHeader.startsWith('Bearer ') ? 
      authHeader.slice(7) : null;

#### 5. فرض المصادقة (Middleware)
```javascript
const makeRequireAuth = (getUserFromToken) => 
  (req, res, next) => {
    const authHeader = req.headers.authorization || '';
    const token = authHeader.startsWith('Bearer ') ? 
      authHeader.slice(7) : null;
    const user = token ? getUserFromToken(token) : null;
    
    if (!user) {
      return res.status(401).json({ error: 'غير مصرح بالدخول' });
    }
    req.user = user;
    next();
  };
// يرفض الطلب إذا لم يكن المستخدم مصرح
```

## 4. ملف توليد التذاكر (lib/pdf-utils.js)

### الوظيفة
توليد ملفات PDF للتذاكر تتضمن رمز QR

### الكود التفصيلي:
```javascript
const generateTicketPdf = (ticketData, qrFile, pdfFile) => {
  return new Promise((resolve, reject) => {
    // إنشاء مستند PDF بحجم A6 (بطاقة صغيرة)
    const doc = new PDFDocument({ size: 'A6', margin: 20 });
    
    // فتح ملف للكتابة والربط مع المستند
    const stream = fs.createWriteStream(pdfFile);
    doc.pipe(stream);

    // إضافة العناوين والنصوص
    doc.fontSize(16).text('تذكرة فعالية', { align: 'center' });
    doc.moveDown();
    doc.fontSize(12).text(`الفعالية: ${ticketData.eventTitle || ''}`);
    doc.text(`النوع: ${ticketData.ticketType || 'عام'}`);
    
    // تفريق بين التذاكر الافتراضية والحضورية
    if (ticketData.isVirtual) {
      doc.text('تذكرة افتراضية للبث المباشر');
    } else {
      doc.text(`المقعد: ${ticketData.seatCategory} رقم ${ticketData.seatNumber}`);
    }
    
    doc.text(`رمز التذكرة: ${ticketData.ticketCode}`);
    doc.moveDown();

    // إضافة صورة رمز QR إذا كانت موجودة
    if (qrFile && fs.existsSync(qrFile)) {
      try {
        doc.image(qrFile, { fit: [150, 150], align: 'center' });
      } catch (imgErr) {
        console.error('Unable to embed QR in PDF:', imgErr);
      }
    }

    // إنهاء المستند وحفظه
    doc.end();

    // انتظار اكتمال العملية
    stream.on('finish', resolve);
    stream.on('error', reject);
  });
};
```

### خطوات العملية:
1. **إنشاء مستند PDF** بحجم A6 (بطاقة)
2. **إضافة المعلومات**: اسم الفعالية، النوع، المقعد
3. **إضافة رمز QR**: لتسهيل التحقق عند الدخول
4. **حفظ الملف**: في النظام

## 5. العمليات الأساسية

### عملية الحجز (Booking Flow):
```
1. المستخدم يختار تذاكر من الفعالية
   ↓
2. التحقق من التوافر (هل عدد التذاكر متوفر؟)
   ↓
3. حجز مقعد مؤقت (Temporary Reserve)
   ↓
4. إنشاء جلسة دفع Stripe
   ↓
5. المستخدم يدخل بيانات البطاقة
   ↓
6. معالجة الدفع والتحقق من النتيجة
   ↓
7. إذا نجح: حفظ الحجز، توليد QR
   ↓
8. توليد PDF وإرسال في البريد الإلكتروني
```

### عملية المصادقة:
```
1. إدخال اسم المستخدم وكلمة المرور
   ↓
2. البحث عن المستخدم في قاعدة البيانات
   ↓
3. مقارنة كلمة المرور (bcrypt)
   ↓
4. توليد Token فريد
   ↓
5. تخزين Token في الجلسة (Memory)
   ↓
6. إرسال Token للعميل
   ↓
7. العميل يرسل Token في رؤوس الطلبات
```

## 6. معايير الأمان المطبقة

### 1. تشفير كلمات المرور
```javascript
// عند التسجيل
const hashedPassword = await bcrypt.hash(password, 10);

// عند التحقق
const isValid = await bcrypt.compare(password, hashedPassword);
```

### 2. المصادقة بالتوكن
- كل طلب يحتوي على رمز Bearer Token
- يتم التحقق من صحة الرمز قبل السماح بالعملية

### 3. الصلاحيات المختلفة
- **Admin**: يمكنه إدارة الفعاليات والمستخدمين
- **Normal User**: يمكنه الحجز والدفع فقط

### 4. معالجة الدفع الآمنة
- استخدام Stripe Elements (لا ترسل بيانات البطاقة للخادم)
- التحقق من الدفع عبر Webhooks

## 9. استكشاف الأخطاء

### الخطأ: "Unable to open database"
- تحقق من صلاحيات المجلد `db/`
- تأكد من وجود المساحة الكافية على القرص

### الخطأ: "Unauthorized"
- تحقق من وجود التوكن في رؤوس الطلب
- تحقق من أن التوكن لم ينته صلاحيته

### الخطأ: "Payment failed"
- تحقق من مفاتيح Stripe
- تأكد من اتصالك بالإنترنت
- استخدم أرقام اختبار Stripe
