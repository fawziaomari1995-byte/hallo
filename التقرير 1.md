# التقرير 1

هذا تقرير موجز يشرح هيكل المشروع ووظائف الملفات والمجلدات الأساسية، بالإضافة لطريقة تشغيل الاختبارات المحلية.

---

## نظرة عامة

المشروع عبارة عن تطبيق ويب لإدارة وعرض الفعاليات مبني على Node.js وExpress مع قاعدة بيانات SQLite. يحتوي على واجهة أمامية في مجلد `public/` وخادم API في `server.js`.

## الملفات الرئيسية

- `server.js`:
  - الخادم الرئيسي المبني على Express.
  - يعرّف مسارات API للتسجيل، تسجيل الدخول، نسيان/إعادة تعيين كلمة المرور، إدارة المستخدمين، إدارة الفعاليات، التذاكر، والتحليلات.
  - يهيئ قاعدة بيانات SQLite ويقدّم الملفات الثابتة من `public/`.

- `db.js`:
  - تهيئة اتصال SQLite المستخدمة عبر التطبيق.

- `lib/server-utils.js`:
  - دوال مساعدة للمصادقة وتوليد التوكنات.
  - ميدلوير `makeRequireAuth` و`makeRequireAdmin` لاستخدامها في حماية المسارات.

- `lib/pdf-utils.js`:
  - منطق توليد ملفات PDF للتذاكر.

## المجلدات

- `public/`:
  - ملفات الواجهة الأمامية (HTML, CSS, JS) لواجهة المستخدم.

- `uploads/`:
  - تخزين الملفات المرفوعة مثل صور الفعاليات.

- `db/`:
  - يحتوي ملف قاعدة البيانات `events.db` المستخدم محليًا.

- `__tests__/`:
  - مجموعة اختبارات مُصمَّمة لتشغيل المشروع محليًا بدون إطار اختبار خارجي مثل Jest.
  - يحتوي `run-tests.js` الذي يستدعي باقي ملفات الاختبار.

## صور من الوظائف والواجهات (مختصر)

- إدارة المستخدمين (`/api/admin/users`):
  - `GET` لعرض المستخدمين (يتطلب صلاحية المدير).
  - `POST` لإنشاء مستخدم جديد (يمكن تحديد `role` و`isActivated`).
  - `PUT` لتعديل الحقول المسموح بها فقط (`role`, `isActivated`).
  - `DELETE` لحذف مستخدم مع منع حذف حساب المدير نفسه.

- المصادقة:
  - `/api/register`, `/api/login`, `/api/forgot-password`, `/api/reset-password`.
  - البريد الإلكتروني يُرسل عبر `nodemailer` (قابل للاستبدال أو المحاكاة في الاختبارات).

- الفعاليات والتذاكر:
  - استعلام الفعاليات، تفاصيل الفعالية، تعليقات، حجز التذاكر، وخريطة المقاعد.
  - دعم فعاليات هجينة (بث مباشر) وتذاكر افتراضية.

## الاختبارات

- لتشغيل الاختبارات المضمنة محليًا (لا تحتاج تثبيت Jest):

```bash
node __tests__/run-tests.js
```

- الاختبارات تغطي:
  - مساعدات الخادم (`lib/server-utils.js`).
  - توليد PDF للتذاكر.
  - تدفقات المصادقة (تسجيل، نسيت كلمة المرور، إعادة تعيين).
  - مسارات الإدارة مع حالات الأذونات (شاملة: إنشاء/تعديل/حذف مستخدمين، فحص أذونات المستخدم العادي مقابل المدير).

## ملاحظات تشغيلية

- قاعدة البيانات الافتراضية مخزنة في `db/events.db` ويمكن حذفها لإعادة تهيئة البيانات الاختبارية.
- لإرسال بريد إلكتروني حقيقي، اضبط متغيرات البيئة `SMTP_HOST`, `SMTP_USER`, `SMTP_PASS`، أو عيّن `mailTransporter` في `server.js`.
- لتشغيل الخادم:

```bash
npm install
npm start
```

ثم افتح:

```
http://localhost:3000
```

## اقتراحات مستقبلية

- تثبيت إطار اختبار مثل `jest` أو `mocha` لتشغيل الاختبارات بطريقة منظمة وتوفير تقارير مفصلة.
- إضافة دعم فعلي للفلترة والصفحات (`q`, `page`, `limit`) في `GET /api/admin/users` إن رغبت ببحث فعلي على مستوى الخادم.
- توثيق واجهات API كاملة بصيغة OpenAPI/Swagger لسهولة الاستهلاك.
